- Muted/secondary text: `#475569` (`text-slate-600`) — never lighter than slate-600 in light mode
- Placeholder text: `gray-400` — acceptable for input placeholders only
- No body text in slate-400 or lighter outside of placeholder context

- Toggle trigger: ghost button with ChevronDown/ChevronUp icon — `text-sm text-slate-500 hover:text-slate-700`

- Textarea: `w-full min-h-[120px] px-4 py-3 border border-slate-200 rounded-xl text-sm font-mono resize-y bg-slate-50`
- Collapsed by default

### Loading / Processing State

- Processing overlay on button: `opacity-50 cursor-not-allowed` + spinner (Loader2 icon from lucide-react, `animate-spin`)
- Inline progress for deep transcription: `text-sm text-slate-500` + progress percentage
- Success toast: AlertState component with success variant

### Error State

- Inline below triggering element (no modal for form errors)
- Style: `text-sm text-red-700 mt-1` using `--ui-color-danger-text`
- For upload/API errors: AlertState component with danger variant

## Interaction Contracts

### Workflow Card Selection (MeetingLandingPage)

- Single click navigates immediately to `/reporter`, `/specialist`, or `/officer`
- No intermediate confirmation step
- Cards filtered by `user.workflowGroups` — users with 1 group see 1 card
- Language selector value is stored in each workflow page independently (not passed as prop)

### Step Wizard Progression

- Linear forward-only by default (Upload → Phiên âm → Thông tin → Kết quả)
- "Bỏ qua" on form step skips to summarize with empty info (uses placeholder prompt)
- No back navigation within a workflow session (reload returns to step 1)
- Step header shows completed/active/upcoming state, not clickable for navigation

### Form Auto-Save

- All 3 info forms auto-save to localStorage on `onChange` (300ms debounce via `useEffect`)
- Key: `meeting_info_draft` (D-09 — single shared key)
- Discriminator: `_type: 'reporter' | 'specialist' | 'officer'` written alongside data
- On load: check `_type` matches current workflow; return null and show empty form if mismatch (silent, no warning)

### Transcription Mode Toggle (Basic / Deep)

- Rendered as 2 pill-style radio buttons: "Nhanh" (Basic) and "Chi tiết" (Deep)
- Active pill: `bg-[#1E3A8A] text-white`
- Inactive pill: `bg-white border border-slate-200 text-slate-700`
- Deep mode shows progress percentage inline

### AI Summary Trigger

- "Tổng hợp" button only enabled after transcription is complete (non-null transcription state)
- Button is disabled + `opacity-50` while summarizing
- Output renders inline below form in same step container (D-15)

### Hover Feedback Timing

- Cards: `transition-colors duration-200`
- Buttons: `transition-colors duration-200`
- All interactive elements: minimum `cursor-pointer`
- Focus states: `focus:ring-2 focus:ring-[#1E40AF]/30` visible in keyboard nav

### Error Recovery

- No `window.alert()` calls — all errors shown as inline AlertState components
- API errors show AlertState with retry guidance
- Form validation errors show `text-sm text-red-700 mt-1` inline below field

## Accessibility

- All form inputs have visible `<label>` elements with `for` attribute linkage
- Color is never the only state indicator (step completion uses icon + color)
- Workflow cards are `<button>` elements (not `<div>` with onClick)
- Focus order follows visual reading order (top-left to bottom-right)
- Language selector is native `<select>` for screen reader compatibility
- Transitions respect `prefers-reduced-motion`: wrap `transition-*` classes where animation is non-trivial
