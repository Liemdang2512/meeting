## Component Specifications

### 1. Email Chips Input (MeetingInfoForm — Step "Thông tin cuộc họp")

**Location:** After the participants section, before the action buttons.

**Structure:**
```
[Label: "Người nhận email"] [counter: "0/20"]
[Input field with placeholder]
[Tag cloud: chip chip chip]
[Error message row — conditional]
```

**Chip anatomy:**
- Container: `inline-flex items-center gap-1 px-2 py-1 bg-indigo-50 text-indigo-700 text-xs font-medium rounded-lg`
- Delete button: `w-4 h-4 text-indigo-400 hover:text-red-500 cursor-pointer transition-colors` — uses an `×` (×) character or X icon at 12px
- Invalid chip (shown briefly before rejection): `bg-red-50 text-red-600 border border-red-200 rounded-lg` — chip flashes and is removed, error message shown below input

**Input field:**
- Class: `w-full px-4 py-3 border-slate-200 focus:border-slate-200 bg-white focus:outline-none text-sm font-medium transition-colors border rounded-xl`
- Placeholder: `Nhập email rồi nhấn Enter hoặc dấu phẩy...`
- Triggers tag creation on: `Enter` key, `,` character, `Tab` key, field blur (if non-empty)

**Validation:**
- On trigger: strip whitespace, validate RFC 5322 simplified regex (`/^[^\s@]+@[^\s@]+\.[^\s@]+$/`)
- If invalid: do NOT create chip; show error below input: `"Email không hợp lệ: [entered value]"`
- If duplicate: do NOT create chip; show: `"Email này đã có trong danh sách"`
- If at limit (20): do NOT create chip; show: `"Đã đạt tối đa 20 địa chỉ email"`
- Error message style: `text-xs font-medium text-red-500 mt-1`

**Limit counter:** `text-xs font-medium text-slate-400` — shows `{count}/20`, turns `text-amber-500` at 18+, `text-red-500` at 20

**Draft behavior:** Recipients are NOT saved to localStorage draft. Field resets on new session. Source: CONTEXT.md decision.

## Interaction States

### Chips Input — Keyboard Flow

1. User types email address in input
2. On `Enter`, `,`, or `Tab`: validate → if valid and not duplicate and under limit → create chip, clear input
3. On invalid: clear input, show error message for 3 seconds then auto-dismiss
4. On `Backspace` with empty input: remove last chip

### Send Button — Loading State

- Button becomes disabled + shows `Spinner` component (existing) at 16px replacing the mail icon
- Text changes to "Đang gửi..."
- No cancel option (fire-and-forget UX)

### Post-Send Success

- Button text changes to "Gửi lại" (same indigo style — resend is a valid action)
- Status row appears below button: "Đã gửi thành công đến {N} địa chỉ" in emerald-600
- Card subtitle updates to "Đã gửi • {N} địa chỉ"
- State resets if user navigates away and returns (no persistence of send state)

### Post-Send Error

- Button switches to red-50/red-700 variant with "Thử lại"
- Status row: "Gửi thất bại. Vui lòng thử lại."
- Clicking "Thử lại" retries with same recipients and subject — transitions back to loading state
