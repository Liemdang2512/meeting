1. **server/routes/admin.ts** — Add two new endpoints BEFORE `export default router`:

   **GET /api/admin/email-settings** (requireAuth + requireAdmin):
   - Query: `SELECT key, value FROM public.admin_settings WHERE key IN ('resend_api_key', 'resend_from_email', 'max_email_recipients')`
   - Return: `{ resend_api_key: string (masked — show only last 4 chars, prefix with "re_..."), resend_from_email: string, max_email_recipients: number }`
   - If no rows exist, return defaults: `{ resend_api_key: '', resend_from_email: '', max_email_recipients: 20 }`

   **PUT /api/admin/email-settings** (requireAuth + requireAdmin):
   - Body: `{ resend_api_key?: string, resend_from_email?: string, max_email_recipients?: number }`
   - For each provided field, UPSERT into `public.admin_settings`:
     ```sql
     INSERT INTO public.admin_settings (key, value, updated_at)
     VALUES ($key, $value, NOW())
     ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value, updated_at = NOW()
     ```
   - Validate max_email_recipients: integer between 1 and 100
   - If resend_api_key is empty string or not provided, skip that field (don't overwrite existing)
   - Return `{ ok: true }`

   **DB table** — Create table inline with first request pattern (or add to schema). Add auto-creation at the top of GET handler:
   ```sql
   CREATE TABLE IF NOT EXISTS public.admin_settings (
     key TEXT PRIMARY KEY,
     value TEXT NOT NULL,
     updated_at TIMESTAMPTZ DEFAULT NOW()
   )
   ```

1. **server/routes/email.ts** — Create new Express router:

   ```typescript
   import { Router } from 'express';
   import { requireAuth } from '../auth';
   import sql from '../db';
   import { buildEmailHtml } from '../../lib/emailHtmlTemplate';
   import { markdownToHtml } from '../../lib/minutesDocxExport';
   ```

   **POST /api/email/send-minutes** (requireAuth):
   - Request body type:
     ```typescript
     {
       subject: string;
       recipients: string[];
       meetingInfo: {
         companyName: string;
         companyAddress: string;
         meetingDatetime: string;
         meetingLocation: string;
         participants: Array<{ name: string; title?: string; role?: string }>;
       };
       minutesMarkdown: string;
     }
     ```
   - Validation:
     - `recipients` must be non-empty array of strings
     - Each recipient must match `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
     - `subject` must be non-empty string
     - `minutesMarkdown` must be non-empty string
     - Read `max_email_recipients` from admin_settings (default 20). Reject if recipients.length > limit.
   - Read `resend_api_key` and `resend_from_email` from `admin_settings` table
   - If resend_api_key is empty/missing: return 500 `{ error: "Chua cau hinh Resend API key. Lien he admin." }`
   - If resend_from_email is empty/missing: return 500 `{ error: "Chua cau hinh email gui. Lien he admin." }`
   - Convert minutesMarkdown to HTML using `markdownToHtml(minutesMarkdown)`
   - Build email body using `buildEmailHtml({ ...meetingInfo, minutesHtml })`
   - Call Resend API directly via fetch (no SDK needed — simpler):
     ```typescript
     const response = await fetch('https://api.resend.com/emails', {
       method: 'POST',
       headers: {
         'Authorization': `Bearer ${apiKey}`,
         'Content-Type': 'application/json',
       },
       body: JSON.stringify({
         from: fromEmail,
         to: recipients,
         subject: subject,
         html: htmlBody,
       }),
     });
     ```
   - Note: PDF attachment deferred — Resend API supports attachments but `downloadAsPdf` uses browser `window.open` + `window.print()` which is client-side only. PDF generation on server would require a headless browser or different library. For now, send HTML email only. The CONTEXT.md mentions PDF attachment but the existing `downloadAsPdf` function cannot run server-side.
   - If Resend returns 2xx: return `{ ok: true, count: recipients.length }`
   - If Resend returns error: parse error body, return 502 `{ error: "Gui email that bai: [resend error message]" }`
