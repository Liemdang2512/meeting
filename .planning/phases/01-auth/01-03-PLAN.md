**Redirect URI constant (use everywhere)**  
- `const apiPublicBase = (process.env.API_PUBLIC_URL ?? \`http://localhost:${process.env.PORT ?? '3001'}\`).replace(/\\/$/, '');`  
- `const googleRedirectUri = \`${apiPublicBase}/api/auth/google/callback\`;`  
- `new OAuth2Client(GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, googleRedirectUri)` — **same** `redirectUri` in authorize query and in `getToken(code)`.

**`GET /api/auth/google`**  
- If `GOOGLE_CLIENT_ID` or `GOOGLE_CLIENT_SECRET` missing, return **`503`** JSON `{ error: "Đăng nhập Google chưa được cấu hình" }`.  
- Build `state` with CSRF protection: random nonce; sign with **HMAC-SHA256** using `process.env.API_JWT_SECRET` **or** short-lived JWT `{ purpose: 'oauth', nonce }` via existing signer — pick one, document in code comment.  
- Set **httpOnly** `oauth_state` cookie **or** signed state only — must validate on callback.  
- Redirect **`302`** to `https://accounts.google.com/o/oauth2/v2/auth` with `client_id`, `redirect_uri=${encodeURIComponent(googleRedirectUri)}`, `response_type=code`, `scope=openid%20email%20profile`, `state=...`, optional `prompt=select_account`.

**`GET /api/auth/google/callback`**  
- Validate `state` (cookie/signature).  
- `const client = new OAuth2Client(clientId, clientSecret, googleRedirectUri);`  
- `const { tokens } = await client.getToken(code);` — if `!tokens.id_token`, return **`502`** or **`400`** JSON with generic error (no raw token in logs).  
- `const ticket = await client.verifyIdToken({ idToken: tokens.id_token, audience: clientId });`  
- `const payload = ticket.getPayload();` — read `sub`, `email`, `email_verified` (boolean).  
- If `!payload?.sub || !payload.email`, return **`400`**.  
- If `payload.email_verified !== true`, return **`403`** JSON `{ error: "Email Google chưa được xác minh" }` (exact string for tests).

**User mapping (default-safe linking — 01-RESEARCH Open Questions)**  
1. If row exists with `google_sub = sub` → **login** that user (ensure `email_verified_at` set if null).  
2. Else if **no** row with that `email` → **create**: `password_hash` NULL, `email_verified_at = now()`, `google_sub = sub`, `profiles` + `ensureFreeMonthlyAllowanceInTx` same transaction pattern as `POST /register`.  
3. Else if row with same `email`, `google_sub IS NULL`, **`password_hash IS NULL`**, and Google verified → **link only in this edge case**: `UPDATE auth.users SET google_sub = sub, email_verified_at = COALESCE(email_verified_at, now()) WHERE id = ...`.  
4. Else if row with same `email`, **`password_hash IS NOT NULL`**, `google_sub IS NULL` → **do NOT auto-link** (account takeover risk per RESEARCH): return **`409`** JSON `{ error: "Tài khoản này đã đăng ký bằng email và mật khẩu. Vui lòng đăng nhập bằng mật khẩu." }` — use this **exact** string for tests and grep.  
5. Else if email exists with **different** non-null `google_sub` not equal to `sub` → **`409`** `{ error: "Email đã gắn với tài khoản khác" }` (keep or align with existing copy).  
6. Google unverified already handled above.

**Session (cross-plan contract with 01-04)**  
- After success: `ensureFreeMonthlyAllowance`, `signToken` / `signRefreshToken`, same cookies as `POST /login`, **`302`** to `${APP_URL}/meeting` (strip slash). Browser-only flow.

**Rate limiting:** Optional on callback; state validation is mandatory.

**Explicit non-goal:** Do **not** implement fetch-only `POST oauth2.googleapis.com/token` + `GET userinfo` as the primary path; library handles protocol details (`aud`, `iss`, clock skew).
