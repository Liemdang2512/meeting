**A. New file `server/lib/sendVerificationEmail.ts`**  
- Export async function `sendVerificationEmail({ to, verifyUrl }: { to: string; verifyUrl: string })`.  
- If `!process.env.RESEND_API_KEY`, log warning and return `{ sent: false }` (register still succeeds; dev without Resend).

- HTML body: short Vietnamese copy + clickable link to `verifyUrl` (full URL).

**B. Env / URL building (canonical: `API_PUBLIC_URL` only)**  
- Build base: `const apiPublicBase = (process.env.API_PUBLIC_URL ?? \`http://localhost:${process.env.PORT ?? '3001'}\`).replace(/\\/$/, '');`  
- Verification link: `verifyUrl = \`${apiPublicBase}/api/auth/verify-email?token=${rawToken}\`` (must never use a separate `API_PUBLIC_BASE` name).  
- Add `API_PUBLIC_URL` to `.env.example` (e.g. `http://localhost:3001`) — document that production must be the HTTPS public origin of the Express API.  
- Do not introduce any other env var for this purpose; all code comments and docs must say `API_PUBLIC_URL`.

**C. `POST /api/auth/register` (server/routes/auth.ts)**  
- After insert, set `email_verified_at` explicitly to **NULL** for self-service row (omit in INSERT or `NULL`).  
- Generate raw token: `crypto.randomBytes(32).toString('hex')`; store **sha256** (hex) in `auth.email_verification_tokens` with `expires_at = now() + interval '24 hours'`, `used_at` NULL.  
- Invalidate or delete previous unused tokens for that `user_id` before insert (optional: single active token).  
- Call `sendVerificationEmail`; do **not** set cookies; do **not** return `token`.  
- Response: **`201`** body **exact shape** `{ ok: true, message: "Đã gửi email xác nhận. Vui lòng kiểm tra hộp thư." }` (adjust wording slightly OK but keep Vietnamese and meaning).

**D. `GET /api/auth/verify-email`**  
- Query: `token` (raw hex from email).  
- Lookup row by `token_hash = sha256(token)`; check `expires_at > now()`, `used_at IS NULL`.  
- On success: `UPDATE auth.users SET email_verified_at = now() WHERE id = user_id`; set `used_at = now()` on token row.  
- Redirect **`302`** to `process.env.APP_URL` + `/login?verified=1` on success (use `APP_URL` already in `.env.example`). On invalid/expired token, redirect to `/login?verified=0` or `/login?verifyError=1` (pick one query key and use consistently).  
- Do not issue JWT on this route (user logs in with password after verify).

**E. `POST /api/auth/login`**  
- Extend SELECT to include `u.email_verified_at`.  
- After bcrypt success, if `email_verified_at == null`: return **`403`** JSON `{ error: "Vui lòng xác nhận email trước khi đăng nhập." }` (exact string for tests).  
- Do not set cookies before this check.

**F. `POST /api/auth/refresh`**  
- After loading user by `payload.userId`, if `email_verified_at` is null: clear cookies, return **`401`** JSON `{ error: "Email chưa được xác nhận" }` (exact string).

**G. `server/routes/admin.ts` POST `/users`**  
- Extend INSERT into `auth.users` to set `email_verified_at = NOW()` (and `google_sub` omitted/null) per CONTEXT (admin-created users verified immediately).
