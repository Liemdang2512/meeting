<objective>
Implement output-token-based billing: charge user AFTER Gemini responds, based on actual output tokens at $3/1M rate.

Purpose: billing reflects real AI cost — user pays for what was actually generated, not a fixed amount.
Output: post-call charge in gemini.ts + balance floor pre-check + wallet display in UI.

Key design decision (differs from original D-04):
- NO charge-at-start / NO auto-refund mechanism
- YES: balance floor check BEFORE call to avoid wasting API on broke users
- YES: charge based on candidatesTokenCount AFTER response
</objective>
